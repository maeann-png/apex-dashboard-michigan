# LeafLink Sales Dashboard

Same architecture as the Apex dashboard: a scraper pulls sales data from the
**LeafLink API**, saves it to `sales_data.json`, the dashboard renders that file,
GitHub Actions refreshes it on a schedule, and GitHub Pages hosts it.

The big difference from Apex: LeafLink has an official API, so instead of grabbing
a browser session cookie that expires every few weeks, you use a long-lived
**Application API key**. Set it once and it keeps working.

```
leaflink-dashboard/
├── scraper.py              ← pulls orders from the LeafLink API
├── dashboard.html          ← the dashboard UI (added after first data pull)
├── sales_data.json         ← generated by scraper (committed so Pages can read it)
├── requirements.txt
├── .env.example            ← copy to .env and add your key
└── .github/workflows/
    └── scrape.yml          ← scheduled GitHub Action
```

---

## 1. Get your LeafLink API key

1. In LeafLink go to **Settings → Applications**.
2. Click **Create an App** (don't reuse the existing Salesforce / Nimbus keys —
   those belong to other integrations).
3. Name it something like `Chill Sales Dashboard`, add your email.
4. Under **Permissions**, set **Orders = Read** (and Products = Read,
   Customers = Read). Never Read & Write — the scraper only ever reads.
5. Save, then **copy the key immediately** — LeafLink won't show the full key again.

The auth scheme is `Authorization: App <your-key>`.

---

## 2. Add the key as a GitHub secret

In the repo: **Settings → Secrets and variables → Actions → New repository secret**

- Name: `LEAFLINK_API_KEY`
- Value: the key you copied

Never paste the key into any file you commit.

---

## 3. Run locally (optional, to test)

```bash
pip install -r requirements.txt
cp .env.example .env        # then paste your key into .env
python scraper.py
```

You should see pages of rows being pulled, ending with `Saved -> sales_data.json`.

---

## 4. Host on GitHub Pages

1. Push this repo to GitHub.
2. **Settings → Pages → Build and deployment → Source: Deploy from a branch**,
   branch `main`, folder `/ (root)`.
3. Your dashboard will be at `https://<username>.github.io/<repo>/dashboard.html`.

---

## 5. Keep it fresh

The included workflow (`.github/workflows/scrape.yml`) runs every 15 minutes,
re-pulls the data, and commits the updated `sales_data.json`. You can also trigger
it manually from the **Actions** tab, or via cron-job.org hitting the
`workflow_dispatch` endpoint (same setup as the Apex project).

---

## Notes

- **CORS:** LeafLink blocks browser requests to its API, so the dashboard can't
  call LeafLink directly — that's why the scraper writes a JSON file the page reads.
  (Same pattern as Apex.)
- **Rate limit:** 300 requests/minute. The scraper backs off automatically on 429s.
- **Trailing slashes:** LeafLink 404s on the wrong slash convention. If an endpoint
  errors, toggle the trailing `/` in `LEAFLINK_ENDPOINT`.
